# 3.1.1 Sample Hello World

*This page last changed on 13 Mar 2007 by u930di.*

## Sample hello-world type program

### Source

```java
import nacaLib.batchPrgEnv.BatchProgram;
import nacaLib.program.Paragraph;
import nacaLib.varEx.DataSection;
import nacaLib.varEx.Var;
public class TestHelloWorld extends BatchProgram
{
    DataSection WorkingStorage = declare.workingStorageSection();
    Var varHelloWorld = declare.level(1).var();
    Var hello = declare.level(5).picX(5).var();
    Var space = declare.level(5).picX(1).value(" ").filler();
    Var world = declare.level(5).picX(5).var();
    public void procedureDivision()
    {
        move("Hello", hello);
        move("World", world);
        perform(DisplayHelloWorld);
        stopRun();
    }

    Paragraph DisplayHelloWorld = new Paragraph(this){public void run(){DisplayHelloWorld();}};void DisplayHelloWorld()
    {
        display(varHelloWorld);
    }
}
```

### Explanations

```java
public class TestHelloWorld extends BatchProgram
```
This program is a batch program as it derives from BatchProgram class.

The working storage section is declared by

```java
DataSection WorkingStorage = declare.workingStorageSection();
```
This declaration is mandatory.

The program's variables are declared as

```java
Var varHelloWorld = declare.level(1).var(); Var hello = declare.level(5).picX(5).var(); Var space = declare.level(5).picX(1).value(" ").filler(); Var world = declare.level(5).picX(5).var();
```
Note that the space leading variables of level 5 is optional. It's set here only for easy reading of the source code.  
Declaration of all variables is always the same, and follows the syntax:

```java
Var <variable name> = declare.level(<level>).<options>.var();
```
a filler is declared as a variable, but the last keyword is filler() instead of var() :

```java
Var <filler name> = declare.level(<level>).<options>.filler();
```
The program entry point is here the method:

```java
public void procedureDivision()
```
It could be also a paragraph ou a section.

The procedure division contains various sentences that act upon the variables. Here we have moves and a paragrpah perform.

```java
move("Hello", hello);
move("World", world);
perform(DisplayHelloWorld);
stopRun();
```
Note that the paragraph is identified by a compiler validated name :  
`DisplayHelloWorld`, not a string that the compiler could not validate ("DisplayHelloWorld").  
The paragraph delcaration is a little bit strange:

```java
Paragraph DisplayHelloWorld = new Paragraph(this){public void run(){DisplayHelloWorld();}};void DisplayHelloWorld()
```
This syntax is imposed by the fact that a paragraph is a java "functor" that can be called indirectly by it's name. It's name can also be registered internally.

In the sample, the paragraph internal code does only

```java
display(varHelloWorld);
```
This displays the "Hello World" message on the java console output.  
Note that as varHelloWorld is build up with 2 variables and a filler, and then can be assambled by chunks.
